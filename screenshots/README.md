# GridOps Screenshots

This directory contains screenshots documenting the key pages and features of the GridOps platform.

## Pages Documented

1. **Login Page** - User authentication interface with demo credentials
2. **Dashboard** - Summary metrics for assets and incidents
3. **Asset List** - Paginated table of utility assets with filtering
4. **Asset Detail** - Individual asset information with telemetry data
5. **Incident List** - Active incidents and tickets
6. **Create Incident** - Form for creating new incidents
7. **Incident Detail** - Full incident information with status transitions
8. **Swagger UI** - API documentation and testing interface

## Screenshot Details

All screenshots capture the GridOps platform as viewed with an Admin user (kgarcia) to show the complete feature set including assignment panels and detailed audit information.

- **Login**: Demonstrates the centered login card with GridOps branding
- **Dashboard**: Shows summary cards for Total Assets, Assets Offline, Open Incidents, and Critical Incidents
- **Assets**: Displays the full paginated asset table with Type and Status filter dropdowns
- **Asset Detail**: Shows individual asset with live telemetry from the SOAP mock system
- **Incidents**: Lists all active incidents in the system
- **Create Incident**: Form interface for creating new incident tickets
- **Incident Detail**: Complete incident with lifecycle state machine (OPEN → ASSIGNED → IN_PROGRESS → RESOLVED → CLOSED) and audit trail
- **Swagger**: API documentation for the GridOps REST endpoints
